# 员工打卡情况管理系统

## 项目简介
**员工打卡情况管理系统** 是一个旨在实现企业员工考勤管理数字化、自动化的信息管理系统。系统通过对员工打卡记录、班次安排、请假类型等数据进行集中管理和深度分析，取代传统低效的手工登记方式，为企业提供精准、高效的考勤管理解决方案。

## 技术栈
### 前端
- **核心框架**: Vue 3
- **UI 组件库**: Element Plus
- **开发语言**: HTML, CSS, JavaScript
- **HTTP 库**: Axios
- **构建工具**: Vue CLI / Vite

### 后端
- **核心框架**: Spring Boot
- **数据层框架**: MyBatis
- **架构模式**: MVC 分层架构
- **开发语言**: Java 8
- **项目管理**: Maven

### 数据库
- **数据库管理系统**: MySQL 8.0+
- **访问工具**: SQLyog / 原生客户端

## 核心功能模块
系统主要实现了三大核心模块的完整数据管理功能：
1.  **班次管理** (`shifts`)
    - 班次信息的增删改查
    - 支持分页与条件查询
2.  **打卡记录管理** (`clock_records`)
    - 员工打卡记录的CRUD操作
    - 按员工、时间、地点等多维度查询
3.  **请假类型管理** (`leave_types`)
    - 请假类型及其规则配置
    - 扣薪策略与证明要求管理

## 快速启动

### 环境准备
1.  **Java环境**: JDK 1.8+
2.  **数据库**: MySQL 8.0+，创建名为 `employee_attendance` 的数据库
3.  **Node.js**: 12.x+
4.  **Maven**: 3.6+

### 部署步骤
1.  **数据库初始化**
    ```sql
    -- 执行项目中的 SQL 脚本
    -- employee_attendance.sql (基础表结构)
    ```

2.  **后端启动**
    ```bash
    # 进入后端项目目录
    cd backend
    # 使用 Maven 打包
    mvn clean package
    # 运行 Spring Boot 应用
    java -jar target/your-project.jar
    ```

3.  **前端启动**
    ```bash
    # 进入前端项目目录
    cd frontend
    # 安装依赖
    npm install
    # 启动开发服务器
    npm run dev
    ```

4.  **访问系统**
    - 前端地址: `http://localhost:8080` (默认)
    - 后端API: `http://localhost:7070`


## 项目特点
- **前后端分离架构**: 采用前后端分离设计，降低耦合度，便于独立开发和部署
- **RESTful API设计**: 后端提供标准化接口，支持多种客户端调用
- **组件化开发**: 前端采用Vue单文件组件，提高代码复用性和可维护性
- **响应式界面**: 基于Element Plus构建，提供良好的用户体验
- **完整的数据操作**: 实现从数据表创建到业务逻辑的全流程开发
